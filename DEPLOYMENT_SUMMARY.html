🚀 LEGALIZE DREAMS - Render Deployment

Professional Single-Deployment Architecture: Django + React on Free Tier

✅ Architecture Decision

Recommendation: PostgreSQL (Database) + Cloudinary (Images)

💾 Database: PostgreSQL

  • Persistent across redeploys
  • Free tier: 256MB storage
  • Cost: $7.99/month
  • Manages: Products, users, orders
  • Data NEVER lost

🖼️ Images: Cloudinary

  • 10GB free storage
  • 25M transformations/month
  • Auto-optimization (50-80% smaller)
  • CDN included (fast delivery)
  • Images ALWAYS available
Why NOT SQLite? Render's filesystem is ephemeral (deleted on each redeploy). All database files lost. PostgreSQL = persistent, reliable alternative.

📊 Single Deployment Architecture

GitHub Push ↓ Render Build (runs build.sh) ├─ React: npm run build ├─ Copy dist/ → Django static/ ├─ Install dependencies ├─ Run migrations └─ Collect static files ↓ Render Service (ONE service runs both) ├─ Django (REST API + Admin) │ └─ Listens on :10000 │ ├─ Serves /api/* (API endpoints) │ ├─ Serves /admin/ (admin panel) │ └─ Serves / (React SPA) │ ├─ PostgreSQL (data persistence) │ └─ Stores products, users, orders │ └─ Cloudinary (media persistence) └─ Stores product images

Why This Works:

  • ✅ Simpler: One service, one deployment
  • ✅ Cheaper: No API latency, no coordination
  • ✅ Faster: No CORS overhead, same origin
  • ✅ Images persist: Cloudinary survives ephemeral storage
  • ✅ Data persists: PostgreSQL manages database

📝 Deployment Checklist

Step 1: Sign up Cloudinary (free)
Go to cloudinary.com → Create account → Note credentials (Cloud Name, API Key, API Secret)
Step 2: Create Render Service
render.com → New Web Service → Connect GitHub → Select LegalizeDreams
Step 3: Add PostgreSQL
While configuring service → "Add PostgreSQL" → Free tier → Copy DATABASE_URL
Step 4: Configure Environment Variables
Add 11 variables to Render dashboard (SECRET_KEY, DATABASE_URL, CLOUDINARY credentials, etc.)
Step 5: Deploy
Render → Deploys → "Deploy Latest Commit" → Wait for "Live" status (3-5 minutes)

🖼️ Image Persistence Guarantee

How Images Survive Redeploys:

  1. User uploads image in admin
  2. Django receives file
  3. django-cloudinary-storage uploads to Cloudinary
  4. URL stored in PostgreSQL database
  5. Frontend renders Cloudinary URL
  6. On redeploy: Database + Cloudinary unchanged = images persist ✅

✅ Data Persists

  • PostgreSQL: Database survives
  • Products, categories, orders: Safe
  • User accounts: Protected

✅ Images Persist

  • Cloudinary: External storage
  • URLs in database: Point to Cloudinary
  • Redeploy: URLs still valid
